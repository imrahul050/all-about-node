# Refresh Tokens

# Introduction

JWT authentication is powerful, but it introduces a challenge:

```text
JWT tokens expire.
```

If a token expires after:

```text
15 Minutes
```

does the user need to log in again every 15 minutes?

In modern applications:

* Gmail
* Facebook
* LinkedIn
* Amazon

users remain logged in for days or months.

This is achieved using:

```text
Access Tokens
+
Refresh Tokens
```

---

# The Problem with JWT

Suppose a JWT expires in:

```text
15 Minutes
```

Flow:

```text
Login
   ↓
Receive JWT
   ↓
Use Application
   ↓
15 Minutes Pass
   ↓
JWT Expired
```

Without a refresh token:

```text
User must login again
```

Poor user experience.

---

# What is a Refresh Token?

A Refresh Token is a special token used to generate new Access Tokens without forcing the user to log in again.

Think of it as:

```text
Access Token = Entry Ticket

Refresh Token = Ticket Renewal Pass
```

---

# Access Token vs Refresh Token

## Access Token

Purpose:

```text
Access APIs
```

Lifetime:

```text
Short
```

Example:

```text
15 Minutes
```

or

```text
1 Hour
```

---

## Refresh Token

Purpose:

```text
Generate New Access Tokens
```

Lifetime:

```text
Long
```

Example:

```text
7 Days
30 Days
90 Days
```

---

# Authentication Flow Without Refresh Token

```text
Login
   ↓
Generate JWT
   ↓
Use APIs
   ↓
JWT Expires
   ↓
Login Again
```

Not ideal.

---

# Authentication Flow With Refresh Token

```text
Login
   ↓
Access Token
+
Refresh Token
   ↓
Access Token Expires
   ↓
Send Refresh Token
   ↓
Generate New Access Token
   ↓
Continue Using Application
```

No login required.

```text
You stay logged in
for weeks
```

Reason:

Google issues:

```text
Access Token
+
Refresh Token
```

behind the scenes.

# Why Not Make Access Tokens Long-Lived?

Interview Question.

Bad Idea:

```text
Access Token
Expires in 1 Year
```

If stolen:

```text
Attacker gets access
for 1 year
```

Huge security risk.

Short-lived access tokens are safer.

# Refresh Token Rotation

Advanced Security Technique.

Flow:

```text
Old Refresh Token
        ↓
Generate New Refresh Token
        ↓
Invalidate Old Token
```

Benefits:

```text
Better Security
```

If an attacker steals an old refresh token:

```text
It becomes useless
```

after rotation.

---

# Logout Flow

When a user logs out:

```text
Remove Refresh Token
```

from storage.

This prevents generating new access tokens.

# Interview Questions

Q: Why do we need Refresh Tokens?

Answer:

Refresh Tokens allow applications to issue new Access Tokens without requiring users to log in repeatedly after short-lived Access Tokens expire.

---

Q: Why keep Access Tokens short-lived?

Answer:

Short-lived Access Tokens reduce security risks because stolen tokens remain usable for only a limited period.

---

Q: What happens when an Access Token expires?

Answer:

The client sends a valid Refresh Token to the server, which verifies it and issues a new Access Token.

---

Q: Can Refresh Tokens expire?

Answer:

Yes.

Refresh Tokens should also have expiration times, although they are typically much longer than Access Tokens.

---

Q: What is Refresh Token Rotation?

Answer:

Refresh Token Rotation is a security technique where every refresh request generates a new Refresh Token and invalidates the previous one.

---

# Senior Interview Answer

Modern authentication systems typically use short-lived Access Tokens and long-lived Refresh Tokens. Access Tokens are used to authorize API requests, while Refresh Tokens are used to obtain new Access Tokens when they expire. This approach balances security and user experience by limiting the impact of stolen tokens while reducing the need for repeated logins.
