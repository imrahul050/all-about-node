# Password Hashing, Encryption and Encoding

# Introduction

One of the most common backend interview questions is:

```text
What is the difference between
Encoding,
Encryption,
and Hashing?
```

Many developers use bcrypt every day but cannot explain why it exists.

As a backend developer, understanding these concepts is critical because they are directly related to user security and authentication systems.

---

# Why Do We Need Data Protection?

Applications store:

* Passwords
* Personal Information
* Payment Information
* Authentication Tokens

If this data is stored incorrectly, attackers can gain access to sensitive information.

Different techniques are used for different purposes.

```text
Encoding
Encryption
Hashing
```

Each solves a different problem.

---

# What is Encoding?

Encoding converts data from one format into another format.

Purpose:

```text
Data Representation
```

not security.

Example:

```text
Hello
```

Base64 Encoded:

```text
SGVsbG8=
```

Anyone can decode it.

---

# Is Encoding Secure?

No.

Encoding is not designed for security.

Anyone who knows the encoding format can decode the data.

Example:

```javascript
Buffer
```

can encode and decode Base64 easily.

---

# Real World Use Cases of Encoding

Examples:

* Base64 Images
* Email Attachments
* URL Encoding
* Data Transfer

Encoding improves compatibility, not security.

---

# What is Encryption?

Encryption converts readable data into unreadable data using a secret key.

Purpose:

```text
Data Protection
```

Unlike encoding, encrypted data can only be read after decryption.

---

# Encryption Flow

```text
Original Data
      ↓
Encrypt
      ↓
Encrypted Data
      ↓
Decrypt
      ↓
Original Data
```

Notice:

Encryption is reversible.

---

# Real Example

Original:

```text
Rahul Kumar
```

Encrypted:

```text
sdf89sdf89sdf89
```

After decryption:

```text
Rahul Kumar
```

returns.

---

# Real World Uses of Encryption

Examples:

* Banking Systems
* Payment Gateways
* HTTPS
* Credit Card Storage
* Government Systems

Whenever original data must be recovered later, encryption is used.

---

# What is Hashing?

Hashing converts data into a fixed-length value.

Purpose:

```text
Verification
```

not recovery.

Most important point:

Hashing is one-way.

---

# Hashing Flow

```text
Password
    ↓
Hash Function
    ↓
Hashed Value
```

Example:

```text
123456
```

becomes:

```text
$2b$10$kjshd8shd8shd...
```

You cannot convert it back.

---

# Why Hashing is Used for Passwords

Imagine storing:

```text
123456
```

directly in the database.

If the database leaks:

```text
All passwords exposed
```

Bad practice.

Instead:

```text
123456
      ↓
Hash
      ↓
$2b$10$......
```

Store only the hash.

---

# Why Not Encrypt Passwords?

Interview Question:

Why not encrypt passwords?

Because encrypted values can be decrypted.

If an attacker obtains the encryption key:

```text
All passwords exposed
```

Hashing eliminates this risk.

---

# What is bcrypt?

bcrypt is a password hashing algorithm.

Purpose:

```text
Secure Password Storage
```

In our project:

```javascript
bcrypt.hash()

bcrypt.compare()
```

are used during registration and login.

---

# Registration Flow

User enters:

```text
123456
```

Before saving:

```javascript
const hashedPassword =
  await bcrypt.hash(
    password,
    10
  );
```

Database stores:

```text
$2b$10$....
```

instead of:

```text
123456
```

---

# Login Flow

User enters:

```text
123456
```

Database contains:

```text
$2b$10$....
```

Verification:

```javascript
await bcrypt.compare(
  password,
  user.password
);
```

Result:

```text
true
```

or

```text
false
```

---

# What is Salt?

A Salt is random data added before hashing.

Purpose:

Prevent identical passwords from generating identical hashes.

Example:

User A:

```text
123456
```

User B:

```text
123456
```

Without salt:

```text
Same Hash
```

With salt:

```text
Different Hashes
```

This increases security.

---

# What are Salt Rounds?

Example:

```javascript
bcrypt.hash(
  password,
  10
);
```

Here:

```text
10
```

is the salt round.

Higher rounds:

```text
More Security
More Processing Time
```

Common values:

```text
10
12
14
```

Most applications use:

```text
10
```

or

```text
12
```

---

# Why bcrypt Instead of MD5?

Older systems used:

```text
MD5
```

Problem:

Too fast.

Modern hardware can crack millions of MD5 hashes quickly.

bcrypt is intentionally slow.

This makes brute-force attacks difficult.

---

# Why bcrypt Instead of SHA256?

SHA256 is excellent for data integrity.

But it was not designed specifically for password storage.

bcrypt includes:

* Salting
* Cost Factor
* Slow Computation

making it better suited for passwords.

# Most Important Interview Questions

Q: Difference Between Encoding, Encryption and Hashing?

Answer:

Encoding converts data into another format and is reversible.

Encryption protects data using a key and is reversible.

Hashing converts data into a fixed-length value and is irreversible.

---

Q: Why Hash Passwords?

Answer:

Passwords are hashed to protect user credentials. Even if the database is compromised, attackers cannot directly recover original passwords.

---

Q: Why bcrypt?

Answer:

bcrypt is specifically designed for password hashing. It uses salting and configurable work factors, making brute-force attacks significantly more difficult.

---

Q: What is Salt?

Answer:

Salt is random data added before hashing to ensure identical passwords produce different hashes.

---

Q: Why bcrypt Instead of MD5?

Answer:

MD5 is extremely fast and vulnerable to brute-force attacks. bcrypt is intentionally slow and more secure for password storage.

---

Q: Can Hashed Passwords Be Decrypted?

Answer:

No.

Hashing is a one-way process and cannot be reversed.

Passwords are verified by comparing hashes rather than decrypting them.

---

# Senior Interview Answer

Passwords should never be stored in plain text. Modern applications use password hashing algorithms such as bcrypt, which combine salting and computational cost factors to protect credentials against brute-force and rainbow table attacks. Unlike encryption, hashing is one-way and is designed specifically for secure password verification.
